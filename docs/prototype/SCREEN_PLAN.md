# FreelanceFlow Prototype Screen Plan

## Recommended prototype scope

Build enough screens to explain the product to the professor and guide implementation. Do not design every edge case first.

FreelanceFlow is now treated as a **web-first SaaS prototype** with long-term portfolio/product ambition. The landing page is the correct first screen because it defines positioning, trust, visual identity, and the value proposition before the dashboard and app flows.

## Priority 1 - Must-have screens

1. Landing / Hero
2. Desktop Dashboard
3. Transactions screen with create panel
4. Mobile Home
5. Mobile Add Transaction

These prove the concept: value proposition, dashboard, and rapid transaction capture.

## Why landing first is correct

- It makes the project presentable as a real SaaS, not just a class exercise.
- It gives the design system a public-facing brand direction.
- It helps explain the problem and value before showing dense financial UI.
- It becomes the first portfolio/GitHub/LinkedIn artifact.

## Priority 2 - Should-have screens

6. Clients and Projects
7. Budgets and Reports
8. Mobile Project Detail
9. Empty State / Onboarding

These show depth and future scalability.

## Priority 3 - Later

10. Login/Register
11. Settings/Profile
12. Account management
13. Detailed report export

## User flows

### Flow A: First-time understanding

Landing -> Ver demo -> Dashboard -> + Movimiento -> Transaction form

### Flow B: Daily mobile capture

Mobile Home -> + Movimiento -> Select Expense -> Enter Amount -> Save -> Success -> Home updated

### Flow C: Client/project tracking

Dashboard -> Clients and Projects -> Project Detail -> Add income -> Project revenue updated

### Flow D: Monthly control

Dashboard -> Budgets -> Category detail -> Warning if over budget

## Design checkpoints

- Does the dashboard communicate financial health in 5 seconds?
- Can a transaction be recorded in under 20 seconds on mobile?
- Can the user understand client/project revenue without reading instructions?
- Is this clearly freelancer finance, not banking/crypto/trading?
- Would the layout be implementable later in HTML/CSS/JS?
