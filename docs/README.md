# Documentación de FreelanceFlow

## Guía de lectura

1. Lea `specs/` para requisitos, decisiones y auditorías oficiales.
2. Continúe con `sdd/` para diseños y planes técnicos.
3. Consulte `prototype/` como referencia visual y handoffs de implementación; no es autoridad del comportamiento final.

## Autoridad y vigencia

El snapshot auditado describe la evidencia congelada en una fecha concreta. Las decisiones posteriores se registran por separado y solo actualizan el punto que declaran; no reescriben retrospectivamente las conclusiones del snapshot.

En caso de conflicto, priorice la fuente oficial de `specs/`, su fecha de corte y las decisiones posteriores registradas explícitamente.
