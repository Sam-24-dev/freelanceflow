# Visión del producto

FreelanceFlow ayuda a profesionales independientes y equipos pequeños a organizar la operación comercial y financiera de su trabajo desde una interfaz web única.

## Problema y valor

La gestión de clientes, proyectos, cobros y gastos suele quedar repartida entre herramientas aisladas. FreelanceFlow reúne esas tareas para facilitar el seguimiento de la operación y de sus cifras principales.

## Usuarios y roles

| Rol | Alcance actual |
| --- | --- |
| Operativo | Gestiona la operación comercial, financiera y de configuración. |
| Administrativo | Consulta únicamente la Bitácora de actividad. |

## Capacidades actuales

| Área | Capacidades |
| --- | --- |
| Seguimiento financiero | Dashboard con indicadores, registro de movimientos, facturas, pagos, reportes y alertas. |
| Gestión comercial | Directorio de clientes, catálogo de servicios, propuestas y proyectos. |
| Configuración | Categorías de gasto, parámetros fiscales estimados, ajustes de facturación y datos básicos de cuenta. |
| Control administrativo | Bitácora de acciones operativas registradas durante la sesión. |

## Flujo operativo

1. Una persona con rol Operativo registra un cliente y define servicios.
2. Crea una propuesta y, al aceptarla, puede convertirla en un proyecto.
3. Gestiona movimientos y facturas del trabajo realizado; los pagos se registran en la pantalla de facturas.
4. Consulta el dashboard, reportes y alertas para revisar la operación.
5. Una persona con rol Administrativo consulta la Bitácora de actividad.

## Límites actuales

FreelanceFlow funciona como un frontend web estático de múltiples páginas. Los datos de ejemplo y los cambios de uso se mantienen en el navegador, por lo que no existe una fuente central compartida entre usuarios o dispositivos.

Los perfiles y sus guardas de navegación son controles de interfaz. No sustituyen autenticación ni autorización reales. Tampoco existen servicios de backend, API, entrega de correo o notificaciones externas, ni procesamiento de pagos.

## Principios de producto

- Priorizar información financiera y comercial visible y accionable.
- Mantener tareas frecuentes cerca de su contexto operativo.
- Usar formularios y estados claros para reducir errores de registro.
